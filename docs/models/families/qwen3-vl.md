# Qwen3-VL (Alibaba, 2026)

> Vision-флагман от Qwen, лучший OCR/document understanding в open-source.

**Тип**: MoE (3B-22B active)
**Лицензия**: Apache 2.0
**Статус на сервере**: скачана (30B-A3B Instruct Q4_K_M + mmproj F16)
**Направления**: [vision](../vision.md)

## Обзор

Qwen3-VL -- специализированная vision-серия от Alibaba 2026. 30B-A3B вариант -- быстрая MoE с 3B активных параметров, как у [qwen3-coder Next](qwen3-coder.md), но с vision. 235B-A22B -- флагман уровня Gemini-2.5-Pro / GPT-5 на multimodal-бенчмарках.

### 30B-A3B Instruct {#30b-a3b}

Рекомендуемая модель для платформы. MoE с 3B active -- скорость как у Qwen3-Coder.

### 235B-A22B {#235b-a22b}

На пределе VRAM (135 GB Q4 + ~3 GB mmproj + контекст). 22B активных -- значительно мощнее в reasoning чем 3B-A3B вариант. Уровень Gemini-2.5-Pro / GPT-5 на multimodal-бенчмарках.

**Ограничения на платформе**:
- Q4_K_M: 135 GB модели + 3 GB mmproj = 138 GB -- помещается в 120 GiB **только впритык** через GTT-overflow
- Реалистично: контекст ограничен ~16-32K
- Скорость: ~22 tok/s (по аналогии с Qwen3.5-122B-A10B)
- Загрузка модели: 3-5 минут

**Когда брать 235B вместо 30B-A3B**:
- Если простые задачи (OCR счёта, описание фото) -- хватит 30B-A3B (быстрее, помещается без проблем)
- Если задача требует **глубокого визуального reasoning** или это **критичный одноразовый запрос** -- 235B даёт результат уровня закрытых флагманов

#### Сложные сценарии где 235B действительно нужна

**1. Reverse engineering сложного UI / системы**

Скриншот корпоративного интерфейса (например ERP-системы или биржевого терминала) с десятками виджетов, графиков, контекстных меню и метрик. 235B может:
- Извлечь иерархию компонентов и их функциональное назначение
- Восстановить data flow между виджетами по визуальным связям
- Предложить упрощённый дизайн с обоснованием каждого решения
- Найти UX-проблемы (несогласованность, скрытые функции, accessibility issues)

30B-A3B справится поверхностно, 235B сделает экспертный аудит.

**2. Анализ научных публикаций с диаграммами**

Загрузить PDF научной статьи с графиками, формулами, таблицами, схемами эксперимента. 235B может:
- Проверить **внутреннюю согласованность** -- соответствие текста графикам, цифр в таблицах формулам
- Найти **методологические проблемы** в схеме эксперимента
- Сравнить с **другой статьёй** (загрузить две одновременно) -- найти противоречия в выводах
- Воспроизвести цифры из графика и пересчитать выводы

Это уровень peer-review.

**3. Forensic-анализ изображения**

Подозрительное фото (документ, удостоверение, сцена). 235B может:
- Найти **признаки фотошопа** (несоответствие теней, перспективы, артефакты сжатия в разных областях)
- Распознать **манипуляции с метаданными** через визуальные несоответствия (например тени не соответствуют времени по EXIF)
- Идентифицировать **несовместимости** (одежда не по сезону для геолокации, исторические анахронизмы)
- Дать аргументированное заключение с указанием конкретных областей подозрения

**4. Перевод документа с сохранением layout**

Сложный документ (договор на корейском, китайская инструкция, арабская брошюра) с:
- Форматированием (списки, таблицы, выделения)
- Подписями к иллюстрациям
- Сносками и ссылками
- Юридическими формулировками

235B может перевести сохраняя **структуру и нюансы**, а не просто текст. Распознаёт устойчивые юридические/технические выражения и переводит идиоматично.

**5. Архитектурный анализ кодбейса по скриншотам**

Серия скриншотов из IDE: дерево проекта, открытые файлы с кодом, output терминала, диаграммы зависимостей в плагине. 235B может:
- Восстановить **архитектуру системы** по visible-частям
- Найти **антипаттерны** (циркулярные зависимости, god-objects, дублирование)
- Предложить **рефакторинг** на уровне конкретных файлов
- Объяснить **почему именно так** не работает (с указанием на проблемные строки в коде)

Это работа сениор-инженера на ревью.

**6. Multi-document compliance check**

Загрузить набор документов (договор, спецификации, регуляторные требования, политики компании) в различных форматах -- PDF, скриншоты Word, фото подписей. 235B может:
- Проверить **взаимную совместимость** документов
- Найти **противоречия** между ними
- Идентифицировать **отсутствующие пункты** для соответствия regulatory requirements
- Сгенерировать структурированный отчёт о найденных issues

**7. Анализ трейдинг-стратегии по графикам**

Скриншоты графиков с разными таймфреймами одного актива (1m, 5m, 1h, 1d) + индикаторы. 235B может:
- Распознать **паттерны** (head-and-shoulders, divergence, support/resistance)
- Найти **соответствие/расхождение** между таймфреймами
- Дать **рекомендацию по входу/выходу** с обоснованием
- Указать **риски** (false breakout, low volume)

Не торговая система, но качественный manual analysis.

**8. Reverse-engineering алгоритма по визуализации**

Анимация/видео работы алгоритма (например визуализация сортировки или ML-обучения). Несколько кадров → 235B:
- Восстановить **код алгоритма**
- Объяснить **сложность по времени и памяти**
- Сравнить с известными алгоритмами
- Найти **оптимизации**

**9. Творческий разбор картины / фотографии**

Фото произведения искусства или композиции. 235B может дать **искусствоведческий анализ**:
- Композиция (правило третей, золотое сечение, диагонали)
- Цветовая палитра и её эмоциональное воздействие
- Стилистическая принадлежность и влияния
- Технические особенности (фокус, экспозиция, постобработка)
- Контекст эпохи и автора (если узнаётся)

**10. Multi-image storytelling и продолжение**

Несколько фотографий → 235B восстанавливает **возможный сюжет**, описывает связи между изображениями, генерирует продолжение истории, или наоборот восстанавливает что было до. Полезно для:
- Сценарной работы
- Исторического анализа фотоархивов
- Crime investigation (по серии фото места происшествия)

## Сильные кейсы

- **OCR на 30+ языках** -- лучшая в open-source. Русский, китайский, японский, арабский в одном потоке
- **Document understanding** -- структурированный JSON-вывод из PDF/счетов/договоров
- **Video understanding** -- ввод нескольких кадров, action recognition
- **Structured output** -- строго валидный JSON по schema
- **Agentic GUI** -- кликабельные координаты на скриншотах
- **Чтение комиксов/манги** -- понимает порядок панелей, текст в баблах
- **UI-to-code** -- скриншот макета → HTML/React

## Слабые стороны

- Reasoning -- через Thinking-вариант (выше latency)
- 235B на пределе VRAM, мало места под контекст

## Идеальные сценарии

- Парсинг чеков/счетов → JSON для бухгалтерии
- Извлечение данных из научных публикаций
- Browser automation: скриншот → следующий клик
- Многоязычный OCR
- Замена платных Document AI сервисов
