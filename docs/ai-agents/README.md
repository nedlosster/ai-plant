# AI-агенты для разработки

AI-агент -- программа, которая автономно выполняет задачи разработки:
читает кодовую базу, планирует изменения, пишет код, запускает тесты,
итерирует по ошибкам. В отличие от автокомплита или чат-бота, агент работает
в цикле (agentic loop) до достижения результата.

## Эволюция: от автокомплита к агентам

```
2021  Автокомплит
      Copilot (preview): дописывает текущую строку/функцию.
      Контекст: открытый файл. Инициатива: разработчик.

2022  Copilot (GA)
      Подсказки по Tab, работа с комментариями как промптами.
      Контекст: открытый файл + соседние вкладки.

2023  Chat + Inline
      ChatGPT, Copilot Chat: диалог о коде, объяснения, рефакторинг.
      Контекст: выделенный фрагмент + промпт. Инициатива: разработчик.

2024  Agentic (ранний)
      Cursor Composer, Aider, Cline: multi-file edits, запуск команд.
      Контекст: проект (частично). Инициатива: смешанная.

2025  Agentic (зрелый)
      Claude Code, Codex, Devin: автономные циклы, sandbox, тесты.
      Контекст: весь проект (100K-1M токенов). Инициатива: агент.

2026  Multi-agent / Team
      Параллельные агенты, специализация (coder/reviewer/tester),
      background execution, 1M+ контекст, bounded autonomy.
      Рынок: $12.8B (рост с $5.1B в 2024).
```


## Ключевые концепции

### Agentic loop

Основное отличие агента от чат-бота -- цикл:

```
Задача
  |
  v
Планирование (разбиение на шаги)
  |
  v
+---> Выполнение шага (правка файла, запуск команды)
|       |
|       v
|     Наблюдение (вывод команды, ошибки, тесты)
|       |
|       v
|     Оценка: цель достигнута?
|       |
|    нет|     да
+-------+      |
                v
            Результат
```

Агент итерирует до достижения цели или исчерпания бюджета.
Чат-бот отвечает один раз и ждёт следующий промпт.


### Context window

Объём информации, которую агент "видит" одновременно:

```
Размер        | Что помещается               | Примеры
--------------|------------------------------|------------------
8K токенов    | 1-2 файла                    | Ранние модели
32K           | 10-20 файлов                 | GPT-4 (2023)
128K          | Средний проект               | GPT-4 Turbo
200K          | Большой проект               | Claude 3.5
1M            | Весь monorepo                | Claude Opus 4.6, Gemini
```

Больше контекст -- лучше агент понимает проект, меньше ошибок
из-за недостатка информации. Но стоимость растёт линейно.


### Tool use

Агент не только генерирует текст -- он вызывает инструменты:

```
Инструмент         | Назначение
-------------------|----------------------------------------
File read/write    | Чтение и редактирование файлов
Shell/Bash         | Запуск команд, тестов, сборки
Grep/Search        | Поиск по кодовой базе
Git                | Коммиты, ветки, diff
Web search/fetch   | Документация, API reference
LSP/IDE            | Навигация по коду, рефакторинг
```

Набор инструментов определяет возможности агента.
CLI-агенты (Claude Code, Aider) работают с файловой системой напрямую.
IDE-агенты (Cursor, Junie) используют LSP и IDE-инспекции.
Cloud-агенты (Codex, Devin) работают в изолированном sandbox.


### Sandboxed execution

Агент запускает код в изолированной среде:
- Docker-контейнер (Codex, OpenHands)
- VM (Devin)
- Локальная файловая система (Claude Code, Aider -- доверие разработчику)

Sandbox защищает от деструктивных действий, но добавляет latency.


## Типы агентов

```
Тип            | Интерфейс  | Примеры                    | Плюсы
---------------|------------|----------------------------|----------------------------
CLI            | Терминал   | Claude Code, Aider, Gemini CLI | Быстрый, scriptable, SSH
IDE-встроенный | VS Code и др.| Cursor, Windsurf, Junie   | Визуальный diff, LSP
Облачный       | Веб-UI     | Codex, Devin               | Параллельность, sandbox
Self-hosted    | Локальный  | OpenCode, Hermes           | Privacy, BYO model
```


### CLI-агенты

Работают в терминале, видят файловую систему, запускают команды.
Подходят для серверной разработки, CI/CD интеграции, SSH.

```
Разработчик <--> [CLI Agent] <--> файловая система, shell, git
```

### IDE-агенты

Встроены в редактор, используют LSP, показывают diff inline.
Подходят для визуальной работы, frontend, рефакторинга.

```
Разработчик <--> [IDE + Agent] <--> LSP, файлы, терминал IDE
```

### Облачные агенты

Работают в облаке, в изолированных sandbox-ах.
Подходят для автономных задач, параллельной работы.

```
Разработчик --задача--> [Cloud Agent] --PR--> GitHub
                              |
                        [sandbox: VM + IDE + browser + terminal]
```

## Как выбрать агента

Быстрый ответ:
- **Ежедневная разработка**: [Claude Code](agents/claude-code/README.md) (CLI) или [Cursor](agents/cursor.md) (IDE)
- **Бюджет**: Gemini CLI (бесплатный tier) или [opencode](agents/opencode/README.md) + локальная модель
- **Privacy/self-hosted**: [Aider](agents/aider.md) или [opencode](agents/opencode/README.md) + llama-server
- **JetBrains**: Junie или [Continue.dev](agents/continue-dev.md)
- **AWS**: Amazon Q Developer
- **Автономные задачи (PR, bugs)**: Devin или Codex

Подробнее: [selection.md](selection.md) -- decision tree, [comparison.md](comparison.md) -- сравнительная таблица.
