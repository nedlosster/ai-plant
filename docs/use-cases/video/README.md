# AI-генерация видео

Open-source генерация видео из текстового описания (text-to-video, T2V) и из изображения (image-to-video, I2V) на локальном GPU.

## Что такое AI-видеогенерация

Diffusion-модели, обученные на миллионах видеоклипов, генерируют короткие видеоролики (2-21 секунд) по текстовому промпту. В отличие от генерации картинок, видеомодели учитывают временную согласованность: движение объектов, физику, камеру.

Два основных режима:
- **Text-to-video (T2V)** -- генерация видео из текстового описания с нуля
- **Image-to-video (I2V)** -- анимация статичного изображения, генерация движения на основе входной картинки

## Что работает на AMD

Видеомодели работают через PyTorch ROCm. Обязательна переменная окружения:

```bash
export HSA_OVERRIDE_GFX_VERSION=11.5.0
```

Основной инструмент -- ComfyUI с плагинами для видеогенерации (ComfyUI-WanVideoWrapper, ComfyUI-VideoHelperSuite).

Протестированные модели:
- **Wan2.1 1.3B** -- работает, быстрая генерация
- **Wan2.1 14B** -- работает, требует значительного VRAM

Остальные модели (CogVideoX, HunyuanVideo, SVD) -- требуют дополнительного тестирования на данной платформе.

## 96 GiB VRAM -- уникальное преимущество

Большинство consumer GPU имеют 8-24 GiB VRAM. 96 GiB unified memory открывает возможности, недоступные на стандартном оборудовании:

- **Wan2.1 14B в fp16** -- полная загрузка модели (~28 GiB веса + ~20-40 GiB промежуточные тензоры) без квантизации и offloading
- **Высокое разрешение** -- генерация в 720p без tiling и OOM
- **Длинные видео** -- больше кадров за один проход
- **Несколько моделей** -- возможность держать T2V и I2V модели одновременно

На GPU с 24 GiB VRAM Wan2.1 14B не запустится в fp16 -- требуется агрессивная квантизация или offload на CPU, что снижает качество и скорость.

## Ограничения (март 2026)

- Максимальная длительность одного клипа -- 5-21 секунд (зависит от модели и разрешения)
- Нет генерации звука -- видеомодели создают только визуальную часть
- Текст в видео -- нестабильный, артефакты
- Руки, пальцы, лица -- типичные проблемы diffusion-моделей
- ROCm для gfx1151 -- экспериментальный, возможны баги
