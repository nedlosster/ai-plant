## FIM-совместимость

Fill-in-Middle (FIM) -- режим, когда модель вставляет код в середину (между prefix и suffix). Критично для автодополнения в IDE.

**Модели с FIM**: Qwen2.5-Coder (все размеры), Codestral, StarCoder2, Devstral Small 2.

**Модели без FIM** (только chat): Qwen3-Coder-Next, Qwen3.5, DeepSeek-V3.

Для автодополнения нужна FIM-модель. Для chat/агентов -- любая.

## Рекомендации для 120 GiB

### Автодополнение (FIM, быстрый отклик)

| Модель | Квантизация | VRAM | tg tok/s | Качество |
|--------|------------|------|----------|----------|
| Qwen2.5-Coder-1.5B | Q8_0 | ~2 GiB | 120.6 | хорошее |
| Qwen2.5-Coder-7B | Q4_K_M | ~5 GiB | -- | высокое |
| Codestral 25.01 | Q4_K_M | ~14 GiB | -- | отличное |

Рекомендация: **Qwen2.5-Coder-1.5B Q8** -- 120 tok/s, занимает 2 GiB. Или **7B Q4** для лучшего качества.

### Chat / рефакторинг

| Модель | Квантизация | VRAM | tg tok/s | Качество |
|--------|------------|------|----------|----------|
| Qwen3-Coder-30B-A3B | Q4_K_M | ~18 GiB | 86.1 | высокое, быстрая MoE |
| Qwen2.5-Coder-32B | Q4_K_M | ~20 GiB | ~12 | отличное (HumanEval 92.7%) |
| Qwen2.5-Coder-32B | Q8_0 | ~40 GiB | ~7 | максимальное |

При 120 GiB: Qwen2.5-Coder-32B в **Q8_0** (~40 GiB) -- максимум качества. При 96 GiB Q8 оставлял мало места для параллельного FIM-сервера.

### Агентные задачи (SWE-bench)

| Модель | Квантизация | VRAM | SWE-bench | tg tok/s |
|--------|------------|------|-----------|----------|
| Qwen3-Coder-Next 80B MoE | Q4_K_M | ~45 GiB | 70.6% | 53.2 |
| Qwen3.5-27B MoE | Q4_K_M | ~17 GiB | 72.4% | 12.6 |
| Devstral Small 2 24B | Q4_K_M | ~15 GiB | 68.0% | -- |

MoE-модели активируют только 3B параметров на токен -- высокая скорость при большом объеме знаний.

### Два сервера одновременно

120 GiB позволяет держать FIM + Chat + запас для крупных моделей:

```
FIM:  Qwen2.5-Coder-1.5B Q8   ~2 GiB
Chat: Qwen3-Coder-30B-A3B Q4  ~18 GiB
Итого:                          ~20 GiB из 120 GiB
Остаток:                        ~100 GiB
```

Или максимальная конфигурация:
```
FIM:  Qwen2.5-Coder-1.5B Q8   ~2 GiB
Chat: Qwen2.5-Coder-32B Q8    ~40 GiB
Итого:                          ~42 GiB из 120 GiB
Остаток:                        ~78 GiB
```

## Рекомендации по задачам

### AI-агенты (Aider, Cline, SWE-agent)

Агенты выполняют многошаговые задачи: анализ кода, планирование, правки, тесты. Критично: качество reasoning, длинный контекст, tool use.

| Приоритет | Модель | VRAM | Почему |
|-----------|--------|------|--------|
| 1 | **Qwen3-Coder-Next** | ~45 GiB | SWE-bench 70.6%, 256K контекст, 53 tok/s |
| 2 | **Qwen3.5-27B** | ~17 GiB | SWE-bench 72.4%, мультимодальная, быстрая загрузка |
| 3 | **Devstral Small 2** | ~15 GiB | SWE-bench 68.0%, 256K контекст, FIM |

Qwen3-Coder-Next -- лучший выбор при наличии VRAM (45 GiB из 120 -- с запасом). Qwen3.5-27B -- если нужна мультимодальность (анализ скриншотов, диаграмм).

### DevOps / инфраструктура

Задачи: Dockerfile, CI/CD, Ansible, Terraform, shell-скрипты, конфигурации, диагностика логов.

| Приоритет | Модель | VRAM | Почему |
|-----------|--------|------|--------|
| 1 | **Qwen3-Coder-30B-A3B** | ~18 GiB | 86 tok/s, хорошее знание DevOps-инструментов |
| 2 | **Qwen2.5-Coder-32B Q4** | ~20 GiB | HumanEval 92.7%, широкий набор языков |
| 3 | **Qwen3.5-27B** | ~17 GiB | Универсальная, русский для документации |

DevOps-задачи редко требуют длинного контекста (конфиги компактные). Быстрый отклик важнее -- MoE 30B-A3B при 86 tok/s оптимален.

### Проектирование / архитектура

Задачи: проектирование API, выбор паттернов, ревью архитектуры, документация, диаграммы.

| Приоритет | Модель | VRAM | Почему |
|-----------|--------|------|--------|
| 1 | **Qwen3-Coder-Next** | ~45 GiB | Максимум знаний (80B MoE), 256K -- весь проект в контексте |
| 2 | **Qwen3.5-122B-A10B** | ~71 GiB | Лучшее качество reasoning, мультимодальная |
| 3 | **QwQ-32B** | ~19 GiB | Chain-of-thought для сложных архитектурных решений |

Для архитектурных задач важно качество reasoning и объем знаний. 120 GiB позволяет использовать 122B MoE -- при 96 GiB это работало впритык.

### Программирование (написание кода)

Задачи: реализация фич, генерация кода, юнит-тесты, рефакторинг.

| Приоритет | Модель | VRAM | Почему |
|-----------|--------|------|--------|
| 1 | **Qwen2.5-Coder-32B Q8** | ~40 GiB | HumanEval 92.7%, максимум качества в Q8 |
| 2 | **Qwen3-Coder-30B-A3B** | ~18 GiB | 86 tok/s -- быстрая итерация |
| 3 | **Qwen2.5-Coder-14B** | ~9 GiB | Баланс скорости и качества |

При 120 GiB: Qwen2.5-Coder-32B в Q8_0 дает лучшее качество генерации. При 96 GiB Q8 было непрактично с параллельным FIM.

### Отладка / диагностика

Задачи: поиск багов, анализ стек-трейсов, чтение логов, объяснение ошибок.

| Приоритет | Модель | VRAM | Почему |
|-----------|--------|------|--------|
| 1 | **Qwen3-Coder-Next** | ~45 GiB | 256K контекст -- большие логи, стек-трейсы целиком |
| 2 | **Qwen3.5-27B** | ~17 GiB | Мультимодальная -- анализ скриншотов ошибок |
| 3 | **Devstral Small 2** | ~15 GiB | 256K контекст, FIM для inline-фиксов |

Длинный контекст (256K) критичен для отладки -- логи, стек-трейсы, несколько файлов одновременно. Мультимодальность Qwen3.5 полезна для анализа скриншотов UI-ошибок.

