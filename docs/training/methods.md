# Обзор методов training и fine-tuning

Платформа: Radeon 8060S (120 GiB GPU-доступной памяти), ROCm 7.x.

## Training vs Fine-tuning

**Training (pre-training)** -- обучение модели с нуля на больших объемах текста. Требует тысячи GPU-часов, петабайты данных. Не рассматривается для данной платформы.

**Fine-tuning** -- дообучение уже обученной модели на специализированных данных. Адаптация под конкретную задачу, домен или стиль. Реально на single GPU.

## Методы fine-tuning

### Full Fine-tuning

Обновление всех параметров модели. Максимальное качество, максимальные требования к ресурсам.

**Требования к VRAM** (формула для AdamW + FP16):
```
VRAM ~ параметры * 16 байт
(веса FP16: 2B + градиенты FP16: 2B + Adam states: 8B + активации: ~4B)
```

| Модель | Параметры | VRAM |
|--------|-----------|------|
| 1B | 1B | ~16 GiB |
| 4B | 4B | ~64 GiB |
| 7B | 7B | ~112 GiB |
| 12B | 12B | ~192 GiB |

На 120 GiB: full FT до 4-7B (с gradient checkpointing).

**Когда использовать**: максимальное качество критично, малый размер модели (1-4B), достаточно VRAM.

### LoRA (Low-Rank Adaptation)

Замораживание оригинальных весов. Добавление обучаемых low-rank матриц к attention layers.

```
Оригинальный слой: W (заморожен)
LoRA-адаптер: W + A * B
  A: [d_model, rank]    -- обучаемый
  B: [rank, d_model]    -- обучаемый
  rank: 8-64            -- гиперпараметр
```

Обучается 1-5% параметров. VRAM снижается в 4-5x.

| Модель | Параметры | VRAM (LoRA r=16) |
|--------|-----------|-----------------|
| 7B | 7B | ~15 GiB |
| 13B | 13B | ~28 GiB |
| 30B | 30B | ~63 GiB |
| 70B | 70B | ~146 GiB |

На 120 GiB: LoRA до 30-40B.

**Когда использовать**: основной метод для моделей 7B+. Баланс качества и ресурсов.

### QLoRA (Quantized LoRA)

Базовая модель квантизируется до 4-bit (NF4). LoRA-адаптеры обучаются в 16-bit. Экономия VRAM в 4-10x при сохранении 90-95% качества full FT.

| Модель | Параметры | VRAM (QLoRA 4-bit) |
|--------|-----------|-------------------|
| 7B | 7B | ~5 GiB |
| 13B | 13B | ~9 GiB |
| 30B | 30B | ~20 GiB |
| 70B | 70B | ~46 GiB |

На 120 GiB: QLoRA до 70B.

**Ограничение на AMD**: bitsandbytes для 4-bit квантизации нестабилен на gfx1151. Unsloth переключает на 16-bit LoRA при `load_in_4bit=True`. До стабилизации bitsandbytes рекомендуется LoRA 16-bit.

**Когда использовать**: модели 30B+, когда LoRA 16-bit не помещается.

### Гиперпараметры LoRA/QLoRA

| Параметр | Типичное значение | Описание |
|----------|------------------|----------|
| `r` (rank) | 8, 16, 32, 64 | Ранг адаптера. Выше = больше емкость, больше VRAM |
| `lora_alpha` | 16, 32 | Scaling factor. Обычно = 2 * r |
| `target_modules` | q_proj, v_proj, k_proj, o_proj | Слои для адаптации. Больше слоев = лучше качество |
| `lora_dropout` | 0.05-0.1 | Dropout для регуляризации |
| `learning_rate` | 1e-4 -- 2e-4 | Скорость обучения |
| `epochs` | 1-3 | Число эпох. Для малых датасетов 1-2, для больших 3+ |
| `batch_size` | 1-4 | Per-device batch size. Ограничен VRAM |
| `gradient_accumulation` | 4-16 | Эмуляция большого batch size |

### DPO (Direct Preference Optimization)

Alignment модели по предпочтениям пользователя. Не требует reward model (в отличие от RLHF).

Вход: пары (промпт, chosen_response, rejected_response).

```
Датасет:
  prompt: "Объясни квантовую физику"
  chosen: "Квантовая физика изучает..."  (хороший ответ)
  rejected: "Ну, это такая штука..."     (плохой ответ)
```

Требования к VRAM аналогичны LoRA (модель + reference model, но reference заморожена).

**Когда использовать**: после SFT (supervised fine-tuning), для улучшения стиля и безопасности.

### GRPO (Group Relative Policy Optimization)

Вариант RLHF без отдельной reward model. Модель генерирует группу ответов, ранжирует их, обучается на лучших.

Проще DPO: не нужны пары (chosen, rejected) -- модель сама генерирует.

**Когда использовать**: когда нет размеченных пар предпочтений, для reasoning-задач.

## Что выбрать

| Задача | Метод | Модель | VRAM |
|--------|-------|--------|------|
| Быстрый эксперимент | LoRA | 1-4B | 5-15 GiB |
| Адаптация под домен | LoRA | 7-13B | 15-28 GiB |
| Максимальное качество (малая модель) | Full FT | 1-4B | 16-64 GiB |
| Большая модель | LoRA 16-bit | 20-30B | 30-63 GiB |
| 70B модель | QLoRA 4-bit | 70B | ~46 GiB |
| Alignment / стиль | DPO + LoRA | 7-13B | 15-28 GiB |
| Reasoning | GRPO + LoRA | 7-13B | 15-28 GiB |

Рекомендация для начала: **LoRA 16-bit на модели 7-13B**. Стабильно работает на AMD, разумное время training, хорошее качество.
